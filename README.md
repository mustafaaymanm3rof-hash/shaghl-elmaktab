# شغل المكتب — دليل النشر (خطوة بخطوة)

نظام متابعة طباعة وتصميم: عملاء، طلبات شغل، مدفوعات، فودافون كاش. بعد النشر هيبقى ليه رابط ثابت يفتحه أي موظف من موبايله زي أي موقع عادي.

الوقت المتوقع: 15-20 دقيقة، مش محتاج خبرة برمجة.

---

## الخطوة 1: اعمل قاعدة بيانات مجانية على Supabase

1. روح [supabase.com](https://supabase.com) واعمل حساب مجاني (بإيميلك أو GitHub).
2. اضغط **New Project**. اختار اسم للمشروع (مثلاً `shaghl-elmaktab`) وباسورد لقاعدة البيانات (احفظه في مكان آمن).
3. بعد ما المشروع يخلص إعداد (دقيقة أو اتنين)، روح لـ **SQL Editor** من القائمة الجانبية.
4. افتح ملف `supabase-schema.sql` الموجود في المجلد ده، انسخ كل اللي فيه، الصقه في الـ SQL Editor، واضغط **Run**.
   - ده هيعمل جدولين: `customers` (العملاء) و `work_entries` (الشغل)، ويفعّل صلاحيات القراءة/الكتابة.
5. روح لـ **Project Settings > API**. هتلاقي:
   - **Project URL** (شكله `https://xxxxx.supabase.co`)
   - **anon public key** (مفتاح طويل)
   - احتفظ بيهم، هتحتاجهم في الخطوة الجاية.

## الخطوة 2: حط بيانات الاتصال في المشروع

1. في المجلد ده، هتلاقي ملف اسمه `.env.example`. اعمل نسخة منه وسمّيها `.env`.
2. افتح `.env` وحط فيه القيم اللي جبتها من Supabase:
   ```
   VITE_SUPABASE_URL=https://xxxxx.supabase.co
   VITE_SUPABASE_ANON_KEY=eyJhbGciOि...
   ```

## الخطوة 3: ارفع المشروع على GitHub (مطلوب لـ Vercel)

1. اعمل حساب على [github.com](https://github.com) لو مفيش عندك.
2. اعمل Repository جديد (فاضي)، وارفعله محتويات المجلد ده (فيه أدوات زي GitHub Desktop لو مش مرتاح بالـ command line).

## الخطوة 4: انشر على Vercel (استضافة مجانية)

1. روح [vercel.com](https://vercel.com) واعمل حساب (سجل دخول بـ GitHub أسهل حاجة).
2. اضغط **Add New > Project**، واختار الـ Repository اللي رفعته.
3. Vercel هيكتشف إنه مشروع Vite تلقائي. قبل الضغط على Deploy:
   - روح لقسم **Environment Variables** وضيف نفس المتغيرين من ملف `.env`:
     - `VITE_SUPABASE_URL`
     - `VITE_SUPABASE_ANON_KEY`
4. اضغط **Deploy** واستنى دقيقة.
5. هتاخد رابط زي `shaghl-elmaktab.vercel.app` — ده الرابط اللي أي موظف يفتحه من موبايله وهيشتغل زي أي موقع، ويقدر حتى يعمله "إضافة للشاشة الرئيسية" فيبقى شكله زي تطبيق.

---

## ملاحظات مهمة

- **كل الموظفين بيشوفوا نفس البيانات لحظيًا** لأنهم بيتصلوا بنفس قاعدة البيانات.
- الإعداد الحالي مفتوح لأي حد معاه الرابط (مفيش تسجيل دخول لكل موظف). ده مناسب كأداة داخلية بين ناس موثوق فيهم. لو حبيت تضيف باسورد لكل موظف لاحقًا، ده ممكن بإضافة Supabase Auth — قولّي لو عايز ده.
- خطة Supabase وVercel المجانية كفاية جدًا لحجم بيانات محل زي ده.
- لو عايز تجرب المشروع على جهازك الأول قبل النشر: افتح Terminal في المجلد ده وشغّل:
  ```
  npm install
  npm run dev
  ```
